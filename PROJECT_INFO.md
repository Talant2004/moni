# Информация о проекте "Саранча"

## Реализованный функционал

### ✅ Главная страница
- Информация о стадных саранчовых
- Отдельные разделы по мароккской саранче, итальянскому прусу и азиатской саранче
- Навигационные кнопки в шапке сайта

### ✅ Навигация
В верхней части сайта реализованы кнопки:
- **Карты** - переход к интерактивным картам
- **Нашествия** - список крупных нашествий
- **Препараты** - перечень препаратов против стадных саранчовых
- **Методика** - описание методики противодействия
- **О проекте** - информация о грантовом проекте

### ✅ Страница "Нашествия"
- Список нашествий с годами
- Детальные страницы с описанием, фотографиями, ссылками
- Подготовка для анимированных карт (заглушка)

### ✅ Страница "Карты"
- Интерактивная карта Республики Казахстан
- Переключение между административной и спутниковой картой
- Фильтры:
  - Область
  - Район
  - Год (с градацией весна/осень)
  - Прогноз угрозы вспышки
  - Обработанные площади

### ✅ База данных
Локальная база данных SQLite со следующими таблицами:
- `invasions` - нашествия
- `preparations` - препараты
- `map_data` - данные для карт
- `invasion_photos` - фотографии нашествий
- `invasion_links` - ссылки на нашествия

### ✅ Многоязычность
- Русский язык (по умолчанию)
- Казахский язык
- Переключатель языков в шапке сайта

### ✅ Адаптивный дизайн
- Полная адаптация для мобильных устройств
- Высококонтрастный интерфейс
- Крупные кнопки для удобного использования в полевых условиях

### ✅ Доступность
- Все страницы доступны без авторизации
- Готовность для доступа по QR коду
- Высокая контрастность для использования на ярком свету

## Технические детали

### Используемые технологии
- **Next.js 14** - React фреймворк
- **TypeScript** - типизация
- **Leaflet** - библиотека для карт
- **better-sqlite3** - локальная база данных
- **react-i18next** - интернационализация

### Структура базы данных

#### Таблица invasions
- `id` - уникальный идентификатор
- `year` - год нашествия
- `title` - заголовок
- `description` - краткое описание
- `full_text` - полный текст
- `region` - регион
- `created_at` - дата создания

#### Таблица preparations
- `id` - уникальный идентификатор
- `name` - название препарата
- `description` - описание
- `active_substance` - действующее вещество
- `application_method` - способ применения

#### Таблица map_data
- `id` - уникальный идентификатор
- `region` - область
- `district` - район
- `year` - год
- `season` - сезон (весна/осень)
- `latitude`, `longitude` - координаты
- `threat_level` - уровень угрозы
- `treated_area` - обработанная площадь
- `population_density` - плотность популяции

## Как добавить данные

### Через SQL
Можно напрямую работать с базой данных `data/sarancha.db`:

```sql
-- Добавить нашествие
INSERT INTO invasions (year, title, description, full_text, region)
VALUES (2021, 'Название', 'Описание', 'Полный текст', 'Регион');

-- Добавить препарат
INSERT INTO preparations (name, description, active_substance, application_method)
VALUES ('Название', 'Описание', 'Вещество', 'Метод');
```

### Через API (нужно создать)
В будущем можно добавить POST endpoints для добавления данных через интерфейс.

## Запуск проекта

1. Установить зависимости: `npm install`
2. Запустить разработку: `npm run dev`
3. Открыть http://localhost:3000

База данных создастся автоматически при первом запуске.

## Деплой

Проект готов к деплою на:
- **Vercel** (рекомендуется для Next.js)
- **Netlify**
- Любой хостинг с поддержкой Node.js

Важно: Убедитесь, что хостинг поддерживает файловую систему для работы SQLite.
